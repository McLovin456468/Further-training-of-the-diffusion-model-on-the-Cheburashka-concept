## 1. Назначение и цель проекта

Диффузионная модель не понимает, как выглядят ранее неизвестные ей объекты (например, чебурашка). Необходимо дообучить модель на новый концепт, чтобы она могла генерировать изображения с этим объектом в различных сценах.

**Цель проекта:**
- Продемонстрировать, что базовая модель не знает целевой концепт.
- Дообучить модель с использованием **LoRA**.
- Дообучить модель с использованием **Textual Inversion**.
- Сравнить результаты двух методов.
- (Опционально) объединить оба метода и оценить синергию.

## 2. Входные данные

### 2.1. Датасет

- **Состав:** 3 изображения целевого объекта (чебурашка).
- **Формат:** произвольный (JPEG/PNG).
- **Источник:** предоставленные изображения.

### 2.2. Базовая модель

|-----------|----------|
| Модель | `runwayml/stable-diffusion-v1-5` |
| Шедулер | `DDPMScheduler` |

| UNet | `UNet2DConditionModel` |
| Текстовый энкодер | `CLIPTextModel` |
| Токенизатор | `CLIPTokenizer` |

### 3.2. Этап 2: Дообучение модели с LoRA

**3.2.1. Загрузка и заморозка моделей**

| Действие | Описание |
|----------|----------|
| Загрузить `DDPMScheduler`, `UNet2DConditionModel`, `AutoencoderKL` | из `runwayml/stable-diffusion-v1-5` |
| Заморозить веса UNet и VAE | `x.requires_grad_(False)` |

**3.2.2. Добавление LoRA**

| Параметр | Значение |
|----------|----------|
| `init_lora_weights` | `"gaussian"` |
| `target_modules` | `["to_k", "to_q", "to_v", "to_out.0"]` |
| `rank` (r) | 128 (или другое значение) |

Использовать `LoraConfig` из библиотеки `peft` и метод `add_adapter`.

**3.2.3. Параметры обучения**

| Параметр | Значение |
|----------|----------|
| `learning_rate` | 2.0e-05 |
| `max_train_steps` | 1000 |
| `train_batch_size` | 1 |
| `max_grad_norm` | 1.0 |
| `lr_scheduler` | `"constant"` |
| `lr_warmup_steps` | 0 |
| `snr_gamma` | 5.0 |
| `lora_rank` | 128 |

**3.2.4. Требования к реализации**

В коде должны быть описаны (в комментариях или markdown-ячейках):

- назначение VAE в пайплайне;
- как семплируется диффузионное время;
- как зашумляется изображение;
- что идёт на вход основной модели;
- как рассчитывается loss;
- какие дополнения нужны для подсчёта loss.

**3.2.5. Сохранение результатов**

- Checkpoint каждые 500 шагов (или чаще)
- Сохранение чекпоинтов LoRA

### 3.4. Этап 4: Обучение Textual Inversion

**3.4.1. Загрузка компонентов**

- `CLIPTextModel` (текстовый энкодер)
- `CLIPTokenizer` (токенизатор)

**3.4.2. Подготовка**

| Действие | Описание |
|----------|----------|
| Добавить новый токен | `tokenizer.add_tokens(["<cheburashka>"])` |
| Изменить размер эмбеддингов | `text_encoder.resize_token_embeddings(len(tokenizer))` |
| Заморозить все веса всех моделей | кроме эмбеддингов текстового энкодера |

**3.4.3. Особенности обучения**

- На каждом шаге использовать **случайные варианты промптов**:
  - `"a photo of <cheburashka>"`
  - `"a picture of <cheburashka>"`
  - `"a <cheburashka>"`
- После шага оптимизатора возвращать **всем эмбеддингам, кроме обучаемого, их исходные значения**

**3.4.4. Параметры обучения**

| Параметр | Значение |
|----------|----------|
| `learning_rate` | ~1e-3 (можно варьировать) |
| `max_train_steps` | 3000 |
| Сохранение чекпоинтов | каждые 500 шагов |
